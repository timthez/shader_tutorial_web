3D Cube

Introduction

In the last tutorial we used shaders to manipulate a 2d triangle. Although it was interesting most graphics today are done in 3d. In this part of the tutorial we will see how to setup an MVP system to show a 3D cube.

Setting The Program to Display the Cube

When the program first starts up it displays a triangle. To display a cube I have built in a key press listener that will change the Vertex Array Object(VAO) and the Vertex Buffer Object(VBO) to use cube vertices instead of the triangle. To switch between the two press 1 for the triangle and 2 for the cube. If the correct transformations are not setup in the shaders, the objects will not display correctly. Using GluLookAt and GluOtho

First off we are going to project using an orthographic projection. We need to stop using the 2D version and use the 3D version. We can do this simply by replacing gluOrtho2d for this:

gluOrtho(-250,250,-250,250,0.001,1000)

You will notice that we have now set the value back to -250 and 250 as this will place our cube in the middle of the screen. We also should remove the translate and make sure we also scale the cube 250 in the z direction as well. If we compile this now we will have little changed with the cube still residing in the middle as a square.

We could now add the last component of MVP that we ignored last tutorial, gluLookAt. This is just like what we have used in previous versions of OpenGL. Lets att the gluLookAt function to our shader:

mat4 gluLookAt(double eyeX,  double eyeY,  double eyeZ,  double centerX,  double centerY,  double centerZ,  double upX,  double upY,  double upZ){
  vec3 f = normalize(vec3(centerX - eyeX,centerY - eyeY,centerZ - eyeZ));
  vec3 UP = normalize(vec3(upX,upY,upZ));
  vec3 s = cross(f,UP);
  vec3 u = cross(normalize(s),f);

  mat4 mat;

  mat[0] = vec4( s.x, s.y, s.z, 0);
  mat[1] = vec4( u.x, u.y, u.z, 0);
  mat[2] = vec4( -f.x, -f.y, -f.z, 0);
  mat[3] = vec4( 0.0, 0.0, 0.0, 1.0);

  mat4 trans;
  trans[0]= vec4(1,0,0,-eyeX);
  trans[1]= vec4(0,1,0,-eyeY);
  trans[2]= vec4(0,0,1,-eyeZ);
  trans[3]= vec4(0,0,0,1);

  return transpose(mat)*transpose(trans);
}

We can use this function in our vertex shader like this:

void main() {
    Color = VertexColor;
    gl_Position = gluOrtho(-250,250,-250,250,0.001,1000) * gluLookAt(250,100,250,0,0,0,0,1,0)  *translate(0,0,0) * scale(100,100,100) * vec4(VertexPosition.x, VertexPosition.y , VertexPosition.z , 1.0);    
}

What is up with the random triangles?

It turns out our simple cube is more complex than it originally seems. Our cube it actually made up of 12 triangles! For the other triangles I colored them the same as the rest of the face so that it seems to be a cube. This face though clearly shows how the cube is actually made up. So why is it triangles? It turns out using triangles is very efficient. Any planar surface can be represented using triangles. Triangles can also be sorted much easier than other polygons. Since we are using OpenGL and are wanting to work at a lower level we primarily use triangles to represent objects. If we wanted to we could create functions that would build complex objects like spheres and cubes using just triangles.

Perspective Projection

To sum up this part of the tutorial we are going to use a perspective projection to view our cube. Let's add another function for doing so:

mat4 gluPerspective(float fovy,  float aspect,  float zNear,  float zFar){
  mat4 mat;
  float f= 1/tan(radians(fovy/2.0));

  mat[0] = vec4(f/aspect,0,0,0);
  mat[1] = vec4(0,f,0,0);
  mat[2] = vec4(0,0,(zNear+zFar)/(zNear-zFar),(2*zNear*zFar)/(zNear-zFar));
  mat[3] = vec4(0,0,-1,0);

  return transpose(mat);
}

Just like in previous versions of OpenGL this function works as expected. Let's use it in place of our gluOrtho function like this:

gl_Position = gluPerspective(60,1,0.001,1000)* gluLookAt(500,200,500,0,0,0,0,1,0)  *translate(0,0,0) * scale(100,100,100) * vec4(VertexPosition.x, VertexPosition.y , VertexPosition.z , 1.0);
